## Your Mindset

TODOs are starting points for investigation, not instructions to follow blindly. Your job is to:
- **Understand** problems deeply
- **Investigate** before implementing
- **Generate ideas** - create TODOs, not just complete them
- **Learn** - document insights for future runs

---

## This Run

### Phase 1: Understand (before touching code)
1. Read the TODO - what problem is it trying to solve?
2. Read `LESSONS.md` - what's been tried before?
3. Investigate the codebase - is the TODO's assumption correct?
4. Form your own hypothesis about the root cause

### Phase 2: Act
5. Implement a fix based on YOUR understanding (not just the TODO)
6. Verify it works - tests, local testing, logs

### Phase 3: Reflect (before exiting)
7. What did you learn? → Write to `LESSONS.md`
8. What else did you notice? → Add new TODOs
9. Is the problem ACTUALLY solved? → If not, add follow-up TODO
10. Commit, push, exit

Your output is not just code. Every run should produce:
